Frequently Asked Questions

If you have any questions about Lyvebyte, please feel free to contact us.
We may have some answers below that will help you out as well.

What is Lyvebyte

Lyvebyte is a real-time IoT cloud platform built on a secure, scaleable serverless architecture, designed to collect event from any web enabled device. Lyvebyte is built with an 'IoT first' infrastructure and written with a modern technology stack all running in the cloud.

Is lyvebyte only for manufacturing?

No it isn't. Lyvebyte grew out of a need for a low-cost production line performance monitoring solution. However, lyvebyte is agnostic when it comes to receiving data.

What customer is this for?

Quite simply, any customer. Anyone looking to push data to the cloud and have real-time feedback in a dashboard is a perfect candidate. Lyvebyte is perfect for manufacturing, agriculture, IoT, etc.

Why lyvebyte?

Lyvebyte works off a subscription model so you can capture where it counts and scale as you need, making it feasible to expand. Lyvebyte utilizes real-time data synchronization which enables all connected clients to be in sync at all times. Lyvebyte doesn’t require any existing automation such as PLC’s or HMI’s, Lyvebyte can be up and running at your facility the same day.

What about an API?

Absolutely, Lyvebyte supports .NET, Node.js, go, Swift, Objective-C, JAVA, C++, and many more.

What is data synchronization?

Instead of typical HTTP requests, Lyvebyte uses data synchronization—every time data changes, any connected device receives that update within milliseconds.

How long is data stored?

We offer different plans ranging from one month and one year to indefinite.

Is my data secure?

Lyvebyte utilizes a flexible, expression-based rules language, to define how data should be structured and when data can be read from or written to. Data can only be accessed through an SSL (Secure Sockets Layer) connection.
